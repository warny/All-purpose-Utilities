# Utils.Fonts — Second quality and security audit pass (2026-07-19)

Follow-up static audit of the current `Utils.Fonts` package after the completed functional fixes and remaining debt recorded in `DONE-2026-07-10.md`. This pass focuses on hostile-font parsing, table-directory invariants, `loca`/`glyf` correctness, cmap state exposure, composite glyph limits, resource ownership, and package stability. No production code is changed.

## Critical and high-priority findings

### 21. Short-format `loca` entries are parsed as signed values

`LocaTable.ReadData` reads short-format offsets through `ReadArray<short>` and shifts each signed value left by one. The OpenType field is `Offset16`, an unsigned 16-bit value divided by two. Encoded values from `0x8000` through `0xFFFF` therefore become negative offsets.

**Risk:** valid large fonts can resolve glyphs before the beginning of `glyf`; malformed fonts can create negative sizes and incidental seek/range failures.

**Fix:** read and write `UInt16`, widen before multiplying by two, and validate monotonicity plus the final offset against the `glyf` table length.

**Priority: P0 binary correctness and hostile-input safety.**

### 22. Short-format `loca` serialization silently truncates invalid offsets

Writing uses `(short)(offset >> 1)` without checking that offsets are non-negative, even, monotonic, or representable in short `loca` format. Odd glyph boundaries lose one byte and offsets above `0x1FFFE` wrap.

**Fix:** validate every offset before writing; automatically switch `head.indexToLocFormat` to long format when necessary, or reject the font with a contextual exception.

**Priority: P0 font corruption.**

### 23. The SFNT directory treats unsigned fields as signed and does not validate table bounds

`numTables` is read as `Int16`; table checksums, offsets, and lengths are read as signed `Int32`. Negative counts, offsets, or lengths are not rejected. Nor is `offset + length` checked with overflow-safe arithmetic against the source length before seeking and allocating.

**Risk:** malformed input can trigger negative allocations, arbitrary seeks, overflow, excessive memory use, or confusing lower-level exceptions.

**Fix:** parse specification-defined unsigned fields as `UInt16`/`UInt32`, apply explicit implementation limits, use checked 64-bit range calculations, and require every table range to fit completely in the font.

**Priority: P0 parser boundary validation.**

### 24. Duplicate directory offsets are silently discarded

Directory records are stored in a `SortedSet<TableDeclaration>`, while `CompareTo` compares only `Offset`. Two records at the same offset compare equal, so one is dropped even if its tag, length, or checksum differs.

**Risk:** the parsed table count no longer matches the file, dependency resolution becomes nondeterministic, and deliberately aliased or malformed entries are hidden instead of validated.

**Fix:** preserve all records in a list sorted by `(offset, originalIndex)` and separately validate duplicate tags, identical ranges, overlaps, and aliases according to an explicit policy.

**Priority: P1 directory integrity.**

### 25. Table overlap, duplicate tags, and directory/header consistency are not validated

The parser ignores `searchRange`, `entrySelector`, and `rangeShift`, accepts duplicate tags, and does not reject table ranges overlapping the directory or each other. `AddTable` later overwrites duplicate tags.

**Fix:** validate the offset-table invariants, tag uniqueness, four-byte alignment where required, range ordering, and overlap policy before parsing any table payload.

**Priority: P1 structural validation.**

### 26. Checksum failures are diagnostics only

A table checksum mismatch only writes to `Debug`; release consumers receive a successfully parsed font with no warning or failure policy. The whole-font checksum adjustment is not validated during reading.

**Fix:** expose strict and permissive parsing modes. Strict mode should reject checksum mismatches; permissive mode should return structured diagnostics attached to the parsed font.

**Priority: P1 integrity semantics.**

### 27. Non-seekable streams are copied without a size limit and the temporary stream is not owned explicitly

`ParseFont(Stream)` copies the complete source into a `MemoryStream` with no maximum length. A hostile or endless stream can exhaust memory. The replacement stream is not disposed deterministically.

**Fix:** accept parser options with a maximum font size, copy incrementally with cancellation in an async overload, and dispose the owned temporary stream after parsing while preserving the caller stream ownership contract.

**Priority: P1 resource exhaustion.**

### 28. Parsing materializes every table before table-specific parsing

Each directory entry is copied into a separate `MemoryStream`, duplicating potentially most of the font in memory in addition to the original byte array or copied source stream.

**Fix:** use bounded slices over the original seekable stream, or copy only when required. Enforce per-table and total allocation budgets.

**Priority: P1 memory amplification.**

### 29. Static `RawReader` and `RawWriter` instances are shared without an explicit thread-safety contract

All font parsing and writing uses shared mutable serializer objects. Their delegate registries and configuration originate outside this package and are not demonstrated to be immutable or concurrency-safe.

**Fix:** make the serializer configuration immutable and document thread safety, or create per-operation reader/writer configurations.

**Priority: P1 concurrency safety.**

### 30. Composite glyph indices are stored as signed `short`

`GlyphCompound.GlyfComponent.GlyphIndex` is a `short` and is read/written as `Int16`, but the glyph index field is unsigned. Fonts with glyph IDs above 32767 become negative.

**Fix:** use `ushort` or a validated `int` throughout and reject component indices outside `maxp.numGlyphs`.

**Priority: P1 glyph correctness.**

### 31. Composite glyph references are not validated for cycles or excessive expansion

The parser limits the number of direct components to 1000, but does not establish a maximum recursive depth, expanded point count, or cycle detection policy when compound glyphs reference other compound glyphs.

**Risk:** cyclic or exponentially expanding component graphs can cause recursion failures, CPU exhaustion, or very large allocations during glyph resolution/rendering.

**Fix:** resolve through a guarded context containing the active glyph stack, maximum depth, maximum expanded components/points, and a deterministic cycle error.

**Priority: P1 hostile-font complexity.**

### 32. Composite glyph length narrows to `short`

`GlyphCompound.Length` accumulates into `int` and returns `(short)size`. Oversized instruction/component data wraps to a negative or smaller length, corrupting `glyf` layout and regenerated `loca` offsets.

**Fix:** keep all table and glyph lengths as checked `int`/`uint`; validate format maxima before serialization and remove narrowing casts from public/internal length contracts.

**Priority: P1 serialization integrity.**

### 33. `loca` offsets are not validated after reading

The table accepts decreasing offsets, negative values, offsets beyond `glyf`, and a final offset inconsistent with the glyph table. Its indexer then returns negative sizes directly.

**Fix:** after loading `loca`, require exactly `numGlyphs + 1` entries, non-decreasing offsets, valid alignment for the selected format, and a final offset within the associated `glyf` length.

**Priority: P1 cross-table invariant.**

### 34. `cmap` uses signed counts and unchecked offsets

`numberSubtables` is read as `Int16` and subtable offsets as signed `Int32`. Negative counts or offsets and out-of-range offset differences are allowed to reach array allocation and `Slice`.

**Fix:** parse unsigned fields, cap subtable count, validate each offset against the cmap table bounds, and calculate lengths using checked arithmetic before creating a slice.

**Priority: P1 untrusted-input parsing.**

### 35. Malformed cmap subtables are silently discarded

`CmapTable.ReadData` catches a broad family of exceptions and only writes to `Debug`. A font can therefore load successfully with all useful mappings removed, and callers cannot distinguish unsupported formats from corruption.

**Fix:** return structured diagnostics and define strict/permissive behavior. Never hide programming exceptions; catch only parser-domain failures with the subtable byte range and format context.

**Priority: P1 observable failure semantics.**

### 36. Mutable arrays expose internal cmap and compound-glyph state

`CmapTable.CMaps` returns its internal array directly, and `GlyphCompound.Instructions` exposes the backing byte array. Callers can mutate parsed state without updating lengths, ordering, checksums, or related tables.

**Fix:** expose immutable/read-only collections and defensive copies, or provide validated mutation APIs that invalidate dependent caches and offsets.

**Priority: P1 state integrity.**

## Medium-priority findings

### 37. Zero-table fonts break derived header properties

`SearchRange` and `EntrySelector` call `BitOperations.Log2((uint)TablesCount)`. A newly constructed font with no tables has no explicit serialization contract and can produce nonsensical values.

**Fix:** reject serialization without required tables, or define zero-table values explicitly before invoking `Log2`.

**Priority: P2 API predictability.**

### 38. Table type discovery depends on unrestricted assembly reflection and unique attributes

The static constructor scans every subclass and calls `Dictionary.Add`. A duplicate table attribute anywhere in the assembly causes type initialization failure for the entire package.

**Fix:** use an explicit registry or source-generated registration, detect duplicates with a precise diagnostic, and validate constructibility at build/test time.

**Priority: P2 startup robustness.**

### 39. Package-wide preview language/features are enabled without a demonstrated need

The project targets `net8.0` but enables preview features and `LangVersion=Latest`, making reproducibility dependent on the installed SDK and broadening consumer compatibility risk.

**Fix:** pin the language version and disable preview features unless a documented public API requires them.

**Priority: P2 packaging stability.**

## Required regression tests

- Read short `loca` values `0x7FFF`, `0x8000`, and `0xFFFF`; verify unsigned byte offsets.
- Reject odd, decreasing, negative, overflowing, and out-of-`glyf` loca offsets; automatically select long format when required.
- Fuzz directory counts, offsets, lengths, duplicate tags, duplicate offsets, overlaps, and arithmetic overflow.
- Verify strict/permissive checksum behavior and structured diagnostics.
- Parse a bounded non-seekable stream and reject input above the configured total size.
- Measure peak allocations to ensure tables are not duplicated unnecessarily.
- Parse and write compound glyph IDs above 32767.
- Reject cyclic and excessively deep compound-glyph graphs.
- Reject wrapped glyph lengths and excessive instruction blocks.
- Fuzz cmap counts/offsets and verify malformed subtables are observable rather than silently lost.
- Prove returned cmap/instruction collections cannot mutate internal state.
- Serialize an empty/incomplete font and assert the chosen explicit contract.

## Deployment warning

Until these findings are addressed, do not parse untrusted fonts in-process without an external file-size limit and execution isolation. Avoid writing fonts that use short `loca` when any glyph boundary is odd or exceeds the representable range, and do not rely on checksum validation or cmap completeness as evidence that a parsed font is intact.
