# Utils.IO — Quality and security audit, pass 2 (2026-07-19)

This pass complements `TODO.md`. It focuses on serialization framing, dynamic reader/writer generation, the source generator, asynchronous stream contracts, and residual gaps left after the completed 2026-07-17 audit. Findings are not fixed by this document.

## High priority

### 17. LEB128 readers accept overlong and overflowing encodings

`ReadULEB128` and `ReadSLEB128` keep reading while the continuation bit is set, with no 10-byte limit for 64-bit values and no validation of the final payload bits. Once `shift` reaches or exceeds 64, C# masks the shift count, so later bytes can overwrite earlier result bits instead of reliably overflowing.

**Risk:** malformed input can consume an arbitrary number of bytes, desynchronize the following fields, and produce a plausible but incorrect value.

**Fix:** limit both encodings to 10 bytes, validate the tenth byte (`0x00`/`0x01` payload rules for ULEB128 and valid sign extension for SLEB128), reject non-canonical overlong encodings when strict canonical input is required, and throw `FormatException` or `OverflowException` distinctly from EOF.

**Priority: P1.**

### 18. Variable-length string prefix width silently defaults to four bytes and can truncate lengths

`ReadVariableLengthString` and `WriteVariableLengthString` treat every `sizeLength` other than `1` or `2` as `4`. The writer casts encoded lengths to `byte` or `ushort` without checking that they fit.

**Risk:** a typo such as `sizeLength: 3` silently selects a different wire format, and long strings written with one- or two-byte prefixes are truncated modulo 256/65536, corrupting the next field boundary.

**Fix:** accept only 1, 2, or 4; reject all other values. Before writing, verify that the encoded byte length fits the selected unsigned prefix. On read, apply a configurable maximum before allocation.

**Priority: P1.**

### 19. Dynamic serializer construction recurses indefinitely for self-referential types

`Reader.CreateReaderFor` and `Writer.CreateWriterFor` recursively create delegates for every attributed member before registering the delegate currently being built. A type containing an attributed member of its own type, or a cycle such as `A -> B -> A`, therefore repeatedly calls `CreateReaderFor`/`CreateWriterFor` until stack exhaustion.

**Risk:** valid recursive object models crash serializer initialization with `StackOverflowException`, which cannot be caught reliably.

**Fix:** detect construction cycles with an explicit in-progress set and reject unsupported recursive by-value layouts with a clear diagnostic. If reference graphs are intended, define an object identity/reference format rather than recursively inlining values.

**Priority: P1.**

### 20. Dynamic reader/writer delegate caches are not concurrency-safe

Both `Reader` and `Writer` store lazily generated delegates in ordinary dictionaries. Concurrent first use of the same or related types can race through lookup, recursive construction, and `Add`, potentially compiling duplicate delegates or throwing because another thread inserted the key first.

**Fix:** either document instances as strictly single-threaded, or protect lookup/construction with a lock and use a cycle-aware cache entry representing `Building`, `Ready`, and `Failed`. Avoid invoking user converters while holding a global lock.

**Priority: P1.**

### 21. Reflection serializers do not validate constructibility and member accessibility before compilation

The dynamic reader always emits `new T()` and assignments to every attributed property/field. Abstract types, interfaces, types without a parameterless constructor, readonly fields, getter-only/init-only properties, and inaccessible members can make expression compilation fail with low-level exceptions. The writer similarly permits write-only/indexed members and getters that are not callable from the generated expression.

**Fix:** validate the full serialization contract before compiling. Report all unsupported members together with declaring type, member name, and reason. Define whether non-public access is a supported feature and test it under trimming/AOT restrictions.

**Priority: P1.**

### 22. Duplicate field orders have no deterministic or rejected contract

Reflection and generated serializers sort only by `FieldAttribute.Order`. Equal orders retain whichever member enumeration order the runtime/compiler happens to provide.

**Risk:** the binary layout can change across runtimes, compiler versions, refactors, or source-generator versus reflection execution.

**Fix:** require unique order values and emit a diagnostic/exception for duplicates. Do not use member name or metadata order as an implicit tie-breaker for a persisted binary format.

**Priority: P1.**

### 23. Source-generator output names collide for homonymous, nested, and generic types

The generator uses `<SimpleTypeName>SerializationExtensions` for the generated class and `<SimpleTypeName>.Serialization.g.cs` for the source hint. Types with the same simple name in different namespaces, nested types, and generic arities can therefore produce duplicate hint names or duplicate generated class/method names.

**Fix:** derive a stable escaped identity from namespace, containing types, metadata name, and generic arity for both hint and class names. Add collision tests with homonymous types in multiple namespaces and nested/generic declarations.

**Priority: P1.**

### 24. Generated code assumes a source shape it does not validate

Generated readers call a public parameterless constructor and assign members directly. The generator does not diagnose abstract/interface types, inaccessible constructors, readonly/init-only/private members, indexers, or types whose containing declarations make the generated extension class inaccessible. It also does not reproduce generic parameters and constraints as part of its generated API design.

**Risk:** consumers receive secondary C# compiler errors in generated files rather than actionable generator diagnostics, and some annotated types cannot be generated correctly.

**Fix:** introduce dedicated diagnostic IDs for every unsupported shape, skip emission after a precise diagnostic, and explicitly design support for generic and nested types before generating them.

**Priority: P1.**

### 25. Custom reader/writer discovery is based on simple names and global symbol search

The source generator searches globally for methods named `Read<Type.Name>` or `Write<Type.Name>`. It does not bind the generated invocation to the discovered method symbol; it simply emits an unqualified extension call with the same simple name.

**Risk:** homonymous types or extension methods in different namespaces can be misdetected, ambiguous, inaccessible, or resolved to a method other than the one that caused `HasCustomReader/Writer` to return true.

**Fix:** resolve and retain the exact method symbol, verify accessibility from the generated context, and emit a fully qualified static invocation. Prefer an explicit registration attribute or well-defined interface over naming convention discovery.

**Priority: P1.**

### 26. Whole-stream helpers remain unbounded despite the previous audit being marked complete

`ReadToEnd` gained a limit, but `ReadToMemoryStream`, `ReadAllText`, and `ReadBlock` still have no size limit. `ReadBlock` also reads byte by byte and stores the complete block before returning.

**Risk:** callers may reasonably believe the completed audit hardened all whole-stream/delimiter helpers, while these APIs can still exhaust memory on untrusted or unexpectedly large input.

**Fix:** add maximum byte/character parameters consistently, make safe bounded overloads prominent, and add cancellation-aware asynchronous variants. Implement delimiter scanning over chunks rather than one `ReadByte` per byte.

**Priority: P1 for untrusted input.**

## Medium priority

### 27. Strict base decoding does not reject every incomplete unpadded quantum

Final validation is centered on whether an additional byte appears to have been produced and on padding count. For encodings without filler, such as base 16, a single symbol leaves four unresolved bits but can reach `Close` without a format error.

**Fix:** define valid terminal symbol counts from `BitsWidth` and require that the final quantum represents an integral number of bytes unless the encoding explicitly defines another rule. Test every possible remainder for base 16, 32, and 64.

**Priority: P2.**

### 28. `Reader.BytesLeft` and `Writer.BytesLeft` require capabilities not guaranteed by their constructors

The constructors accept any stream, but `BytesLeft` accesses both `Length` and `Position`. Non-seekable streams commonly throw `NotSupportedException`. For a writer, “bytes left” is also undefined for expandable streams and can become negative when position exceeds current length.

**Fix:** either expose this property only through a seekable/bounded abstraction, return a nullable/try result, or document and validate the required capabilities at construction. Rename the writer property if it is intended to mean `Length - Position` rather than capacity remaining.

**Priority: P2.**

### 29. `Push(offset, origin)` is not transactional

`Reader.Push` and `Writer.Push` add the current position to the stack before calling `Seek`. If seeking fails, the stack still contains a position that the caller did not intentionally establish.

**Fix:** perform the seek first while retaining the original position locally, then push only after success; alternatively remove the pushed entry in a catch block. Add tests for overflow, invalid origins, and a stream whose `Seek` throws.

**Priority: P2.**

### 30. Stream wrappers do not implement modern span and asynchronous paths explicitly

`PartialStream`, `StreamCopier`, and `BaseEncoderStream` override only array-based synchronous operations. Calls through `Read(Span<byte>)`, `Write(ReadOnlySpan<byte>)`, `ReadAsync`, or `WriteAsync` therefore depend on base-class fallback behavior, which may allocate, execute synchronously, or provide weak cancellation semantics.

**Fix:** implement span overloads directly and async overloads where the underlying resource supports them. Preserve partial bounds, target fan-out error policy, position restoration, cancellation, and disposal semantics across every path.

**Priority: P2.**

### 31. Primitive read APIs disagree on EOF behavior

`IReader.ReadByte` returns an `int` and can report `-1`, but `RawReader.ReadByte` and `ReadSByte` cast that value directly, turning EOF into `255`/`-1`. Other primitive methods call `ReadBytes` using its default non-throwing mode and may receive short arrays, causing unrelated indexing or conversion exceptions.

**Fix:** make primitive deserialization exact by default. Introduce a clear `TryReadByte`/partial-read API separately, and ensure every fixed-size primitive throws `EndOfStreamException` with type and expected-byte context on truncation.

**Priority: P2.**

### 32. Generated and reflection serializers can diverge in converter selection

The reflection path resolves converters through runtime type hierarchy/interfaces and custom delegate registration order. The source generator chooses between a naming-convention method and generic `Read<T>/Write<T>` based on compilation-wide symbol discovery. The same annotated model can therefore produce a different binary layout depending on which path is used.

**Fix:** define one converter-resolution specification and share it between runtime and generator metadata where possible. Add contract tests that serialize the same model through generated and reflection paths and compare exact bytes and error behavior.

**Priority: P2.**

## Required tests

- Reject 11-byte, overflowing, and non-canonical ULEB128/SLEB128 values; verify min/max boundaries.
- Reject invalid `sizeLength` values and over-capacity one-/two-byte string prefixes.
- Detect direct and indirect recursive serialization models without stack overflow.
- Exercise concurrent first-use delegate generation.
- Diagnose abstract, generic, nested, readonly, init-only, inaccessible, indexed, and constructor-less annotated types.
- Reject duplicate `FieldAttribute.Order` values consistently in runtime and generated paths.
- Generate homonymous types in different namespaces and multiple generic arities without collisions.
- Enforce limits on `ReadAllText`, `ReadToMemoryStream`, and `ReadBlock`.
- Test all terminal symbol remainders for base 16/32/64 strict decoding.
- Verify exact EOF exceptions for every primitive type.
- Compare byte-for-byte output and failures between generated and reflection serializers.
- Test span/async wrapper behavior, cancellation, partial bounds, and disposal.
