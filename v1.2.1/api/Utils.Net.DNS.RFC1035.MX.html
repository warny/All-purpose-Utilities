Remarks

Each MX record consists of:

  • Preference: A 16-bit integer indicating priority. Lower values are preferred, meaning a mail transfer agent should try servers in ascending order of Preference.
  • Exchange: A domain name specifying the mail exchange host (e.g., mail.example.com).

For instance, if example.com has two MX records, one with preference 10 and another with preference 20, the sending mail server will first attempt the host with preference 10. If that fails or is unavailable, it tries the host with preference 20.

The DNS library may perform additional processing (e.g., retrieving the A or AAAA address of the Exchange) when resolving MX records.

The RDATA layout defined in RFC 1035 §3.3.9 is:

+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                  PREFERENCE                   |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
/                   EXCHANGE                    /
/                                               /
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+

PREFERENCE is the priority indicator (lower values preferred) and EXCHANGE is a <domain-name> identifying the target mail server. Implementations often follow up with address lookups for the exchange host.
