Remarks

A KEY RR can indicate a zone key (for DNSSEC), an end-entity key, or a user/account key. The Flags field includes bits that govern usage and key ownership, and can also signify the presence of a no-key state or an extension field.

If both ProhibitedForAuthentication and ProhibitedForConfidentiality are set, then this record effectively asserts that there is "no key." This can be used to claim a zone is unsigned (insecure), or to prohibit usage for certain cryptographic functions.

The Protocol and Algorithm fields follow RFC 2535 §3 definitions, and the public key (PublicKey) is algorithm-specific data. For further details on usage, see:

  • RFC 2535 for DNS security extensions
  • RFC 2536 (DSA keys)
  • RFC 2537 (RSA/MD5)
  • RFC 2539 (Diffie-Hellman)

The RDATA is composed of:

  • Flags (16 bits) indicating key usage, key owner type, and extensions.
  • Protocol (8 bits) describing the intended use (DNSSEC, TLS, email, IPSEC, etc.).
  • Algorithm (8 bits) identifying the signing algorithm.
  • Public Key (variable length) containing the algorithm-specific key material.

Gets or sets the cryptographic algorithm. Common values include 3 (DSA), 1 (RSA/MD5), etc. for DNSSEC usage.

public bool Extended { get; set; }

Extension

An optional 16-bit extension field present only if Extended is set in the Flags.

[DNSField(Condition = "Extended")]

Flags

Gets the 16-bit flags field which indicates key usage, key owner type, extension flags, and signatory bits. This is manipulated through properties such as ProhibitedForAuthentication, ProhibitedForConfidentiality, Extended, KeyOwner, and SignatoryField.

[DNSField]
public ushort Flags { get; }

Property Value

ushort

KeyOwner

Gets or sets the key owner (user/account, zone key, host/entity, or reserved). This corresponds to bits 6-7 in the flags.

public KeyOwner KeyOwner { get; set; }

Gets or sets a value indicating whether the key is prohibited for confidentiality use (flag bits 0-1 set to 01 in the specification).

Protocol

Gets or sets the 8-bit protocol value. Typically 3 (DNSSEC), but can be TLS (1), email (2), IPSEC (4), or others as assigned by IANA.

[DNSField]

PublicKey

Gets or sets the raw public key bytes. This is algorithm-specific data whose format depends on the Algorithm.

[DNSField]
public byte[] PublicKey { get; set; }

byte[]

Remarks

If the No-Key bits are set in Flags, the PublicKey should be omitted or ignored.

SignatoryField

Gets or sets the 4-bit signatory field (bits 12-15), indicating whether the key can validly sign updates (RFC 2137).

public byte SignatoryField { get; set; }
