Remarks

An SRV record indicates:

  • Priority: A 16-bit integer indicating the preference for this target server. Lower values are more preferred.
  • Weight: A 16-bit load-balancing weight. If multiple servers have the same Priority, clients should distribute connections proportionally based on these weights.
  • Port: The TCP or UDP port on which the service is provided.
  • Server: The domain name of the target host providing the service.

For example, a typical SRV record might look like: 

_sip._tcp.example.com. IN SRV 10 60 5060 sipserver.example.com.

This indicates that for the service _sip over TCP, example.com uses sipserver.example.com on port 5060 with priority 10 and weight 60.

SRV records facilitate service discovery by allowing multiple servers to be specified, each with its own priority and weight. Clients look up the SRV record to find the hostname and port for the given service, potentially balancing load or choosing higher-priority servers first.
