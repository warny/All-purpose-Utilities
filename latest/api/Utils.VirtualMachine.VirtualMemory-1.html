Remarks

The master process (accessible via MasterProcess) automatically receives a ReadWrite mapping for every page created by AllocatePage(). Additional processes created with CreateProcess() start with an empty page table; use MapPage(VirtualProcess<TAddress>, VirtualPage, TAddress, PageAccess) to grant them access to specific pages.

Thread safety: this class is not thread-safe. All allocations, mappings, and process operations must occur on the same thread. When the same VirtualPage is mapped into multiple processes, concurrent reads or writes from different threads are not synchronized and produce undefined behaviour.

FreePage(VirtualPage, bool)

Frees a physical page, removing all mappings to it from every process in this instance and marking it as freed. Freed pages are removed from Pages and their backing memory is no longer accessible via AsReadOnlyMemory().

public void FreePage(VirtualPage page, bool force = false)

Parameters

page VirtualPage

The page to free. Must have been allocated by this instance.

force bool

When true, removes all mappings in non-master processes before freeing. When false (default), throws InvalidOperationException if the page is still mapped into any non-master process.

Exceptions

ArgumentNullException

Thrown when page is null.

ArgumentException

ObjectDisposedException

Thrown when page has already been freed.

VmInvalidOperationException

Thrown when force is false and the page is still mapped into one or more non-master processes. Unmap the page first or use force: true.
