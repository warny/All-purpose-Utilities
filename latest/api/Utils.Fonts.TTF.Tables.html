Namespace Utils.Fonts.TTF.Tables

Classes

AcntTable

The accent attachment table (tag: 'acnt') provides a space-efficient method of combining component glyphs into compound glyphs to form accents. Accented glyphs are a very restricted subclass of compound glyphs.

AcntTable.AccentDescription

Represents the description of an accented glyph, associating a primary glyph with one or more accent attachments.

AcntTable.AccentDescription.Multiple

Format 1 description: two or more secondary components referenced via the extension sub-table. Stored as 6 bytes: UInt16 primary word (bit15=1), Int32 absolute extension offset.

AcntTable.AccentDescription.Single

Format 0 description: a single secondary (accent) component identified by a direct table index. Stored as 4 bytes: UInt16 primary word, byte attachment point, byte secondary index.

AvarTable

The 'avar' (Axis Variations) table maps normalized design-axis coordinates through per-axis piecewise-linear segment maps, allowing non-linear axis progressions and ensuring that certain canonical normalized coordinates (−1, 0, +1) map to themselves.

AvarTable.SegmentMap

The piecewise-linear segment map for one design axis. Must include mappings for –1→–1, 0→0 and +1→+1 as required by the spec.

BslnTable

The baseline table (tag: 'bsln') defines the positions of baselines for each script in the font, allowing mixed-script text to align glyphs from different writing systems on a common line.

CmapTable

Represents the 'cmap' table which provides the character-to-glyph mapping for a TrueType font. The table contains one or more subtables (each identified by a platform ID and platform-specific ID) that define different mapping formats.

CmapTable.CmapSubtable

Represents a subtable identifier defined by a platform ID and platform-specific ID. Used as a key for the cmap subtables.

CvarTable

The CVT variations table (tag: 'cvar') provides delta adjustments to CVT values for different instances of a variable font. It uses the tuple variation store format: each tuple variation covers one region of the variation space and carries packed deltas for the CVT entries it affects.

CvarTable.TupleVariation

One tuple variation record: covers one region of the variation space and carries the CVT deltas to apply when a font instance falls inside that region.

CvtTable

The CVT (Control Value Table) contains a series of signed 16-bit integers that are used as control values during the hinting process. These values are used by the font program to adjust glyph outlines.

DsigTable

The 'DSIG' (Digital Signature) table contains one or more cryptographic signatures that attest to the authenticity and integrity of a font. Most tools treat this table as an opaque blob: they preserve it on read and discard or regenerate it on write. This implementation stores the entire table as raw bytes after the 12-byte outer header.

FdscTable

The 'fdsc' (Font Descriptors) table provides a set of named scalar values that describe characteristics of the font — such as optical weight, contrast, or size. Each descriptor has a 4-byte tag and a 16.16 fixed-point value.

FdscTable.FontDescriptor

One named descriptor entry: a 4-byte tag and a 16.16 fixed-point scalar value.

FeatTable

The 'feat' (Feature Name) table lists the Apple Advanced Typography (AAT) layout features supported by the font and their available settings. Each feature has a numeric identifier, a name-table entry, and one or more named setting values.

FeatTable.FeatureNameRecord

Describes one AAT layout feature, its flags, its name-table entry, and its available settings.

FeatTable.FeatureSetting

One named setting for a feature (e.g. "On", "Off", "Standard Ligatures").

FmtxTable

The 'fmtx' (Font Metrics) table provides glyph-based typographic metrics: it identifies a reference glyph whose phantom points carry the horizontal and vertical metric data, and stores the indices of those phantom points within the glyph.

FpgmTable

The FPGM (Font Program) table contains a set of instructions (bytecode) that are executed once when the font is loaded. These instructions establish routines used for hinting glyphs.

FvarTable

The 'fvar' (Font Variations) table defines the design axes and named instances for a variable font. Each axis specifies a minimum, default and maximum design-space coordinate, and each named instance represents a specific position in the design space (e.g. "Regular", "Bold", "Light").

FvarTable.InstanceRecord

Represents one named instance (a specific point in the design space).

FvarTable.VariationAxisRecord

Describes one design axis of the variable font.

GaspTable

The 'gasp' table contains information which describes the preferred rasterization techniques for the typeface when it is rendered on grayscale-capable devices at various sizes. Each entry specifies a maximum pixels-per-em threshold and the corresponding rendering behaviour.

GaspTable.GaspRange

Describes the recommended rendering behaviour for one PPEM size range.

GlyfTable

The 'glyf' table contains the data that defines the appearance of the glyphs in the font. It includes the specification of the points that describe the contours that make up each glyph outline as well as the instructions for grid-fitting the glyph. The table supports both simple glyphs and compound glyphs (i.e. glyphs composed of other glyphs).

HdmxTable

The 'hdmx' (Horizontal Device Metrics) table stores pre-computed advance widths for specific pixels-per-em sizes, allowing rasterizers to avoid recomputing them at render time. Each HdmxTable.DeviceRecord covers one PPEM size and contains one advance-width byte per glyph.

HdmxTable.DeviceRecord

Contains per-glyph advance widths (in pixels) for one specific PPEM size.

HheaTable

The 'hhea' table contains information needed to layout fonts whose characters are written horizontally, that is, either left-to-right or right-to-left. This table provides global metrics for the font, including typographic values such as ascent, descent, line gap, and other essential data used during glyph layout.

HmtxTable

The 'hmtx' table contains metric information for the horizontal layout of each glyph in the font. It begins with the hMetrics array, where each entry consists of the advance width and left side bearing for a glyph. The number of long horizontal metrics is specified in the 'hhea' table, and in a monospaced font, only one entry is required (but must still be present).

KernTable

Represents the 'kern' table which contains kerning pairs for adjusting the spacing between glyphs in a TrueType font. This implementation supports the most common format (version 0) with a single horizontal kerning subtable.

LcarTable

The 'lcar' (Ligature Caret) table specifies caret positions within ligature glyphs so that a text caret can be placed at intermediate positions inside a ligature (e.g. between the two characters of a fi ligature). Each ligature glyph has one or more caret values.

LocaTable

The 'loca' table stores the offsets to the locations of the glyphs in the font relative to the beginning of the 'glyf' table. Its purpose is to provide quick access to the data for a particular glyph. For example, in the standard Macintosh glyph ordering, the character A is the 76th glyph in a font. The 'loca' table stores the offset from the start of the 'glyf' table to the position at which the data for each glyph can be found.

LtshTable

The 'LTSH' (Linear Threshold) table stores a per-glyph pixel-per-em threshold value. Below this threshold the glyph is non-linear (i.e. hinting is used); at or above it the glyph scales linearly and hinting can be suppressed for better appearance. A value of 1 means the glyph is always linear; 0 means the threshold is unknown.

MaxpTable

The 'maxp' table establishes the memory requirements for a font. It begins with a table version number and the number of glyphs in the font. The remaining entries set maximum values for various parameters (such as maximum points, contours, and storage), which are used to allocate resources for the font.

OpbdTable

The 'opbd' (Optical Bounds) table provides per-glyph adjustments that shift the effective optical edge of a glyph inward or outward, improving the visual alignment of text at the margins. Each glyph can have four adjustments: left, top, right, and bottom.

Os2Table

The 'OS/2' table contains metrics and other data required for correct rendering on Windows and OS/2 platforms. It is present in virtually every TrueType/OpenType font and describes properties such as weight, width, style, typographic metrics, Unicode character ranges and vendor identification.

PcltTable

The 'PCLT' table contains PCL 5 characterization data: information needed to identify and render the font on PCL 5-capable printers. The table has a fixed size of 54 bytes.

PostTable

The 'post' table contains information needed to use a TrueType font on a PostScript printer. It provides data for the FontInfo dictionary, the PostScript names for all glyphs, and memory usage information for the PostScript driver.

PrepTable

The PREP (Control Value Program) table contains instructions (bytecode) that are executed before rendering any glyph. These instructions typically perform scaling and other pre-processing needed for proper hinting.

PropTable

The 'prop' (Glyph Properties) table assigns directional and typographic property bits to each glyph. Properties include writing direction (LTR/RTL/neutral), hanging punctuation, kerning suppression, and attachment class.

TrakTable

The 'trak' (Tracking) table specifies inter-character spacing adjustments as a function of point size. For each nominal tracking level (e.g. "loose", "normal", "tight") a set of per-size values (in font design units) gives the amount to add to each glyph's advance width.

TrakTable.TrackData

One direction's worth of tracking data (horizontal or vertical).

TrakTable.TrackEntry

One entry in the tracking table: a nominal tracking value and a name-table entry, with per-size spacing adjustments.

VheaTable

The 'vhea' table contains information needed to lay out fonts whose characters are written vertically, that is, either top-to-bottom or bottom-to-top. It is the vertical analogue of the 'hhea' (horizontal header) table and has the same 36-byte fixed structure.

VmtxTable

The 'vmtx' table contains metric information for the vertical layout of each glyph in the font. It is the vertical analogue of the 'hmtx' table. Its layout consists of a series of longVerMetric records (advance height + top side bearing) followed by additional top-side-bearing-only values for the remaining glyphs. The number of long-metric entries is given by NumOfLongVerMetrics, and the total glyph count by NumGlyphs.

Structs

AcntTable.ExtensionEntry

A single entry in the extension sub-table (2 bytes), referencing one secondary glyph attachment. Bit layout: [bit15=isLast | bits14-8=SecondaryInfoIndex(7) | bits7-0=PrimaryAttachmentPoint].

AcntTable.SecondaryEntry

A single entry in the secondary (accent) data table (3 bytes): Int16 secondary glyph index followed by one byte attachment point.

AvarTable.AxisValueMap

A single input→output coordinate mapping entry within a AvarTable.SegmentMap. Both values are F2Dot14 (divide by 16 384 to obtain the real normalized value).

OpbdTable.OpticalBounds

Optical bound adjustments for one glyph (left, top, right, bottom), either in design units (format 0) or contour-point indices (format 1).
