InstructionPointer

Gets or sets the current instruction pointer indicating the byte offset into Data. Setting this to a negative value (typically -1) signals program termination to the VirtualProcessor<T> execution loop.

public int InstructionPointer { get; set; }

Property Value

int

Remarks

Any negative value causes the execution loop to stop and is treated as normal termination. This means stack underflow from CallStack.Return (which also returns -1), explicit Terminate() calls, and erroneous negative jump targets all collapse into the same observable state — all appear as normal completion. Distinguish them at the bytecode or caller level if the difference matters.

Methods

Terminate()

Signals program termination by setting InstructionPointer to -1. The VirtualProcessor<T> execution loop stops before the next instruction. Prefer this over setting InstructionPointer to Data.Length so that intent is explicit and the mechanism can evolve without changing every HALT handler.
