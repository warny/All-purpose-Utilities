GetXPath(XmlElement)

Gets the XPath expression that uniquely identifies the given XmlElement.

public static string GetXPath(this XmlElement element)

Parameters

element XmlElement

The element for which to generate the XPath.

Returns

string

A non-empty XPath string such as /root/parent/child[2]. Positional predicates are included whenever another sibling with the same name exists before or after this element.

Remarks

Generated paths use Name, which includes any document-specific namespace prefix (e.g. /ns:root/ns:child). The returned XPath can only be evaluated correctly when the same prefix-to-URI mappings are registered in an XmlNamespaceManager used at evaluation time.

Elements that have a sibling with the same qualified name — either before or after — always include a positional predicate (e.g. /root/item[1]) to uniquely identify the node.

GetXPath(XmlNode)

Gets the XPath expression that uniquely identifies the given XML node in its document.

public static string GetXPath(this XmlNode node)

Parameters

node XmlNode

The XML node for which to generate the XPath.

Returns

string

A non-empty XPath string for XmlDocument, XmlElement, and XmlAttribute nodes. The document root returns "/".

Remarks

Generated paths use Name, which includes any document-specific namespace prefix (e.g. /ns:root/ns:child). The returned XPath can only be evaluated correctly when the same prefix-to-URI mappings are registered in an XmlNamespaceManager used at evaluation time.

Elements that have a sibling with the same qualified name — either before or after — always include a positional predicate (e.g. /root/item[1]) to uniquely identify the node.

Exceptions

ArgumentNullException

Thrown when node is null.

NotSupportedException

Thrown for node kinds that do not have a meaningful unique XPath representation, such as text, comment, processing-instruction, CDATA, and entity-reference nodes.

ReadChildElements(XmlReader, string?)

Reads and yields the immediate child elements of the current element in the XML reader.

public static IEnumerable<XmlReader> ReadChildElements(this XmlReader reader, string? elementName = null)

Parameters

reader XmlReader

The XML reader positioned at the parent element.

elementName string

Optional name filter. When specified, only child elements with this qualified name are yielded. When null, all immediate child elements are yielded.

Returns

IEnumerable<XmlReader>

An enumerable sequence yielding an isolated XmlReader sub-reader for each immediate child element. The sub-reader is managed by the iterator: partially consumed sub-readers are drained automatically when the enumerator advances to the next child, so consumers need not read or skip remaining content before the next iteration.

Remarks

Depth-based tracking ensures that only elements at exactly one level below the starting element are yielded, regardless of element names at any depth. An empty parent element (<parent />) yields no children without advancing the outer reader past sibling elements.

Reader-state contract: each yielded reader is an independent sub-reader created by ReadSubtree(). When the iterator advances to the next sibling, Dispose() is called on the previous sub-reader, which reads through any unconsumed content and positions the outer reader immediately after the child's closing tag. The consumer must not advance the outer reader directly; only the sub-reader should be used to read the child's content.

Exceptions

ArgumentNullException

Thrown when reader is null.

InvalidOperationException

Thrown when the reader is not positioned at an element node.
