Namespace Utils.VirtualMachine

Classes

BoundedStack<T>

A stack with a configurable maximum depth. Push throws VmLimitExceededException (with OperandStackDepth) when the depth limit is reached; Pop and Peek throw InvalidOperationException when the stack is empty.

CallFrame

Represents a single activation frame on a CallStack, carrying the return address and a per-frame local-variable store keyed by name.

CallStack

A frame-based call stack that stores a CallFrame per subroutine invocation, providing access to per-frame local variables.

CallStackContext

A DefaultContext that carries an ICallStack for processors that implement subroutine calls.

ConditionalBlock

Represents an active if/else conditional block on a ControlFlowStack.

Context

Represents the base execution context used by VirtualProcessor<T>.

ControlFlowContext

A DefaultContext that carries a ControlFlowStack for processors implementing structured control flow (conditionals, loops, try/catch/finally).

ControlFlowStack

Manages the nesting of structured control flow blocks at runtime: conditionals (ConditionalBlock), loops (LoopBlock), and exception handlers (ExceptionBlock).

DefaultContext

A default context implementation providing a bounded object operand stack.

ExceptionBlock

Represents an active try/catch/finally block on a ControlFlowStack.

ExecutionLimits

Immutable execution parameters for a single call to Execute(T, ExecutionLimits, CancellationToken), Run(ExecutionLimits, CancellationToken), or RunAsync(ExecutionLimits, CancellationToken).

FullContext

A DefaultContext combining an ICallStack for subroutine calls with a ControlFlowStack for structured control flow. Use this when the VM needs both call/return and if/loop/try semantics.

InstructionAttribute

Identifies a method as a virtual machine instruction and stores its metadata.

InstructionBudgetExceededException

Thrown when the number of dispatched instructions exceeds the configured instruction budget.

LoopBlock

Represents an active loop on a ControlFlowStack.

MemoryAccessException

The exception thrown when a virtual memory access violates the page's mapped access rights, or when the target address is not mapped or is invalid in the process's page table.

NumberReader

Factory methods for obtaining INumberReader instances.

ScheduledProcess<T>

Wraps a context and dedicated processor pair with scheduling metadata, allowing Scheduler<T> to track lifecycle state, priority, and cooperative yield signals.

Scheduler<T>

A cooperative, priority-based process scheduler that time-slices ScheduledProcess<T> instances, giving each process its own VirtualProcessor<T>.

SimpleCallStack

A lightweight call stack that stores only return addresses, with no per-frame local-variable storage. Prefer this over CallStack when frames do not need local variables.

TypedStackContext<TValue>

A context providing a strongly-typed operand stack, avoiding boxing for value types. Use this instead of DefaultContext when all stack values share a common type (e.g. int for integer VMs, double for floating-point VMs).

VirtualMachineLimits

Immutable structural configuration for a virtual machine instance. Encapsulates capacity limits for stacks, scheduler, and virtual memory.

VirtualMemoryContext<TAddress>

A DefaultContext that associates an instruction stream with a specific VirtualProcess<TAddress>, allowing the executing processor to perform virtual memory reads and writes on behalf of that process.

VirtualMemory<TAddress>

Manages a pool of physical VirtualPage instances and a set of VirtualProcess<TAddress> objects, each with their own virtual address space.

VirtualPage

Represents a fixed-size physical memory page managed by a VirtualMemory<TAddress>. Pages are created exclusively by AllocatePage() and mapped into process virtual address spaces with per-process access rights.

VirtualProcess<TAddress>

Represents a virtual process with its own page table mapping virtual page indices to physical VirtualPage instances and their per-process access rights.

VirtualProcessorException

Represents errors that occur while executing a VirtualProcessor<T>.

VirtualProcessor<T>

Provides an abstract base class for a virtual processor that interprets and executes instructions from a byte array-based context. Instructions are defined by methods annotated with InstructionAttribute.

VmInspectorException

Thrown when an IVmInspector<T> callback raises an exception, allowing callers to distinguish inspector failures from instruction-execution faults.

VmInvalidOperationException

Thrown when a VM operation is performed in an invalid state (e.g. stack underflow, malformed bytecode structure, or exhausted hard identifier counters).

VmLimitExceededException

Thrown when a caller-configured VM resource limit is exhausted.

VmLimitException

Abstract base for all exceptions thrown when a caller-configured VM resource limit is exhausted.

Interfaces

ICallStack

Represents a call stack that tracks return addresses for nested subroutine calls.

IControlFlowBlock

Base interface for all structured control flow blocks that can be pushed onto a ControlFlowStack: conditionals, loops, and exception handlers.

INumberReader

Provides methods to read numbers from a virtual machine Context.

IVmInspector<T>

Optional inspector interface that receives notifications before each instruction is dispatched. Inject an implementation into Inspector to enable step-by-step debugging, profiling, or instruction tracing without subclassing the processor.

PageAccess

Specifies the access rights for a virtual memory page mapping.

ProcessState

Represents the lifecycle state of a process managed by Scheduler<T>.

VmInspectorPhase

Identifies which IVmInspector<T> callback threw.

VmLimitKind

Identifies which configured resource limit was exhausted when a VmLimitExceededException is thrown.
