Remarks

This projection has a single true singularity: the antipodal point (lat=0°, lon=±180°), which is the "point at infinity" for a stereographic projection centered at (0°, 0°) — there is no finite (x, y) that correctly represents it. GeoPointToMapPoint(GeoPoint<T>) does not throw at that point; instead it substitutes T.Epsilon (the smallest positive value representable by T) for the zero denominator, which produces an extremely large but finite k factor and, in turn, extremely large (not NaN, but potentially positive infinity after overflow) x/y values rather than a clear error. Callers that need points near the antipodal meridian should check for this case explicitly (e.g. degree.AreEqual(geoPoint.Latitude, 0, tolerance) && degree.AreEqual(System.Math.Abs(geoPoint.Longitude), 180, tolerance)) rather than relying on the output being finite and meaningful.
