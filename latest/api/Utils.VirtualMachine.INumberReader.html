ReadSLEB128(Context)

Reads a signed LEB128-encoded integer from the supplied context. LEB128 is byte-order independent; this default implementation delegates to ReadByte(Context). A 64-bit value requires at most 10 bytes; longer sequences are rejected as malformed. For the 10th byte (shift = 63), only bit 0 contributes to the result (as the sign bit of long). Bits 1–6 must be consistent with sign extension: all zeros for a non-negative value (0x00) or all ones for a negative value (0x7F). Any other pattern would require more than 64 bits of signed precision and is rejected. Overlong encodings (e.g. zero encoded as 10 bytes) are accepted as long as the total length does not exceed 10 bytes and the 10th byte passes the constraint above.

long ReadSLEB128(Context context)

Parameters

context Context

The execution context containing the instruction stream.

Returns

long

The signed integer decoded from the LEB128 byte sequence.

Exceptions

FormatException

Thrown when the encoding exceeds 10 bytes or the 10th byte carries bits that would require more than 64 bits of signed precision.

Reads an unsigned LEB128-encoded integer from the supplied context. LEB128 is byte-order independent; this default implementation delegates to ReadByte(Context). A 64-bit value requires at most 10 bytes; longer sequences are rejected as malformed. The 10th byte must not have payload bits beyond bit 63 (only the low bit is valid).
