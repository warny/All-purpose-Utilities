Represents the DNS header section of a DNS packet, including standard bits from RFC 1035 §4.1.1 and the AD/CD extensions introduced in RFC 2535. The header provides information about flags, counts for queries/answers/authorities/additionals, and other DNS-level metadata.

Remarks

A DNS header contains:

  • ID: A 16-bit identifier assigned by the program that generates any kind of DNS query.
  • Utils.Net.DNS.DNSHeader.Flags: A collection of DNS flags and codes packed into a 16-bit field (e.g., QR, OpCode, AA, etc.).
  • Counts of queries, answers, authority records, and additional records (see Utils.Net.DNS.DNSHeader.QDCount, Utils.Net.DNS.DNSHeader.ANCount, Utils.Net.DNS.DNSHeader.NSCount, Utils.Net.DNS.DNSHeader.ARCount).

The wire format defined in RFC 1035 §4.1.1 (extended by RFC 2535) is:

+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                      ID                       |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|QR|   Opcode  |AA|TC|RD|RA| Z|AD|CD|   RCODE   |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                    QDCOUNT                    |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                    ANCOUNT                    |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                    NSCOUNT                    |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                    ARCOUNT                    |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+

RFC 2535 introduces the AD (Authenticated Data) and CD (Checking Disabled) bits. Clients should only trust AD when the upstream server is trusted, and the CD flag allows requesting unsigned data for local validation.
