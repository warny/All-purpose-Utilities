FormVariantType

A variant node inside a form-producing element (<Replacement>, <OrdinalException>, <Ordinal>). Plays two roles depending on whether Forms is present:

  • Intermediate node DimensionType + VariantValue add one constraint; child <Variant> elements cascade further constraints.
  • Leaf node DimensionType + Forms provide one output form per dimension value in the order declared by the matching <Dimension> element. Empty entries are skipped.

All (constraints, form) pairs are collected at load time and merged by constraint set into OrdinalVariantRule or VariantRule entries — no runtime changes required.

TriggerReplaceType

A text-replacement rule inside a TriggerType. Selects the most specific matching variant form, or To as the unconditional default. Uses the same FormVariantType nesting as Replacement, Ordinal, and OrdinalException: leaf nodes provide positional forms or a single value; intermediate nodes add dimension constraints via type+variant.

TriggerType
