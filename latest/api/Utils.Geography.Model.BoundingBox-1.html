Remarks

Antimeridian-crossing boxes are not supported. ValidateBoundingBox(GeoPoint<T>, GeoPoint<T>) always orders longitudes with T.Min/T.Max, so a box intended to span from 170°E to 170°W (a 20° arc across the antimeridian) is silently converted into a 340°-wide box instead. Contains(GeoPoint<T>), Intersects(BoundingBox<T>), LongitudeSpan, and GetCenterpoint() all inherit that interpretation and will return incorrect results for such inputs. If your use-case requires global coverage crossing ±180°, split the query into two boxes (e.g. [170°, 180°] and [-180°, -170°]) and union the results.
