Remarks

This class is safe for concurrent use: the Utils.Security.Authenticator._algorithmFactory is invoked on each computation to produce a fresh, privately owned HMAC instance that is disposed immediately after use (#31, #32).

The secret key is copied at construction time and is never exposed through a public property (#30). To export the key for provisioning, call ExportKey().

Parameters

algorithmFactory Func<HMAC>

A factory that returns a fresh, unkeyed HMAC instance on each call. See Utils.Security.Authenticator._algorithmFactory for the full contract. The factory is invoked once at construction time to validate that it produces a non-null instance.

key byte[]

The binary secret key. Must be at least 16 bytes. A defensive copy is made (#30).

digits int

Number of digits, [1, 9] (#29).

intervalLength int

Validity window in seconds. Must be positive (#29).

Authenticator(string, byte[], int, int)

Initializes a new instance of Authenticator using a named HMAC algorithm.

public Authenticator(string algorithmName, byte[] key, int digits, int intervalLength)

Parameters

algorithmName string

The algorithm name resolved through CryptoConfig (e.g. "HMACSHA1", "HMACSHA256"). Throws if the name is blank or does not resolve to an HMAC (#33).

key byte[]

The binary secret key. Must be at least 16 bytes. A defensive copy is made; mutating the original array after construction has no effect (#30).

digits int

The number of digits in the generated code. Must be in [1, 9]. Values above 9 are not supported because the HOTP 31-bit truncation step cannot produce a uniform 10-digit distribution within int range (#29).

intervalLength int

The duration in seconds for which each code is valid. Must be positive (#29).

Exceptions

ArgumentException

algorithmName is null, blank, or does not resolve to an HMAC algorithm, or key is shorter than the minimum length.

ArgumentOutOfRangeException

digits is not in [1, 9] or intervalLength is not positive.

Remarks

The previous factory used HMAC-SHA256, which does not match the default RFC 6238 profile and would produce codes that do not match a normally provisioned Google Authenticator account (#34). Use GoogleAuthenticatorSha256(byte[]) for SHA-256.

Creates an authenticator using HMAC-SHA256, 6 digits, 30-second window. This profile requires explicit provisioning metadata — it does not match the default Google Authenticator TOTP profile (#34).
