Remarks

The operation is transactional: the loop is located first without mutating the stack. The exception is thrown before any block is popped, so the stack remains intact when malformed bytecode issues BREAK outside a loop.

EndFinally(Context)

Handles an ENDFINALLY instruction: if a catch handler is pending (stored by a prior Throw(Context, object?) call when both CatchAddress and FinallyAddress were set), redirects InstructionPointer to that catch handler and clears the pending address, then transitions the block to Catch. When no pending catch is present but an exception is in flight (finally-only block, or the innermost finally in a nested try), pops the block and propagates the exception to the next outer handler by calling Throw(Context, object?) recursively. Otherwise, simply pops the block (normal exit from a try/finally without an exception).

public bool EndFinally(Context context)

Throw(Context, object?)

Handles a THROW instruction: pops all blocks until the nearest ExceptionBlock whose Phase is Try is found, stores the thrown value in ThrownValue, and redirects InstructionPointer according to the following rules:

  • If the block has a finally clause, jumps to FinallyAddress first. When a catch clause is also present, its address is stored in PendingCatchAddress so the ENDFINALLY handler can jump there after the finally block completes. The block transitions to Finally.
  • If the block has only a catch clause (no finally), jumps directly to CatchAddress. The block transitions to Catch.

Blocks already in Finally or Catch are skipped so that a throw from inside a handler body propagates to the next outer try scope instead of re-entering the current handler. The matched ExceptionBlock remains on the stack so that the handler body can read ThrownValue; ENDTRY pops it.

public bool Throw(Context context, object? value)

context Context

The current execution context.
