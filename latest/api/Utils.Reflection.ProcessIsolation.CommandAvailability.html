Exists(string)

Returns true when an executable is present in PATH.

public static bool Exists(string commandName)

Parameters

commandName string

File name of the executable to probe.

Returns

bool

true if the command can be located; otherwise false.

Remarks

On Windows, when commandName has no extension, each extension listed in PATHEXT is also tried (matching how cmd.exe/CreateProcess resolve a bare command name), so Exists("ffmpeg") finds ffmpeg.exe without the caller having to know the platform-specific executable extension.

TryResolve(string, out string?)

Locates an executable in PATH and returns its canonical absolute path.

public static bool TryResolve(string commandName, out string? absolutePath)

Parameters

commandName string

File name or rooted path of the executable to locate.

absolutePath string

When this method returns true, the canonical absolute path to the executable; otherwise null.

Returns

bool

true if the executable was found; otherwise false.

Remarks

Callers should retain the resolved path and pass it directly to process launches rather than re-resolving the bare command name each time. Re-resolving introduces a TOCTOU window: the PATH value or the file at each candidate directory can change between the check and the launch, allowing a different (possibly hostile) binary to be executed.

On Windows, when commandName has no extension, each extension listed in PATHEXT is also tried; see Exists(string) for details.
