Remarks

Coefficient range: not every window is bounded to [0, 1]. Hann, Hamming, and Blackman are (modulo floating-point rounding noise of at most a few ulps right at the edges, where the ideal mathematical value is exactly zero) non-negative and never exceed 1. FlatTop<T>(int), however, is defined by IEC 61672 with coefficients that are negative by design near its edges; callers must not assume a shared [0, 1] range across every window in this class.

Precision: every window is computed using double-precision trigonometry (Cos(double)) regardless of the requested output type T, then converted with T.CreateChecked. This is a deliberate choice, not an accidental precision leak: it gives every output type (including decimal, which has no ITrigonometricFunctions<TSelf> implementation and could not use a generic-precision cosine anyway) the same double-precision-accurate standard window coefficients, rather than constraining T to ITrigonometricFunctions<TSelf> and losing accuracy (and decimal support) for low-precision output types whose own trigonometric implementation would be no more accurate than double's in the first place.

Methods

Blackman<T>(int)

Computes the Blackman window: w[n] = 0.42 − 0.5·cos(2πn/(N−1)) + 0.08·cos(4πn/(N−1)). Very low side-lobes (−58 dB); wider main lobe than Hann. Coefficients lie in [0, 1] (touching 0 at both edges) in exact arithmetic; floating-point rounding can produce a coefficient a few ulps below zero right at the edges.

FlatTop<T>(int)

Computes the Flat Top window (IEC 61672 coefficients): w[n] = 0.21557895 − 0.41663158·cos(2πn/N) + 0.277263158·cos(4πn/N) − 0.083578947·cos(6πn/N) + 0.006947368·cos(8πn/N). Excellent amplitude accuracy; broadest main lobe. Unlike the other windows in this class, coefficients are not confined to [0, 1]: the flat-top shape is negative by design near its edges (down to roughly −0.2).

Hamming<T>(int)

Computes the Hamming window: w[n] = 0.54 − 0.46 · cos(2πn/(N−1)). Reduced side-lobe leakage compared to Hann; does not go to zero at the edges. Coefficients lie in [0.08, 1].

Hann<T>(int)

Computes the Hann (Hanning) window: w[n] = 0.5 · (1 − cos(2πn/(N−1))). Good general-purpose window with −18 dB/octave roll-off. Coefficients lie in [0, 1] (touching 0 at both edges), modulo floating-point rounding noise.
