Remarks

This class is not thread-safe. Concurrent calls to Execute(T, CancellationToken, long) or ExecuteStep(T) on the same instance share a dispatch buffer and must not overlap; use one VirtualProcessor<T> instance per thread, or synchronize externally.

Breakpoints

A set of instruction-pointer addresses that trigger OnBreakpoint(T, int, string) when execution reaches them. Breakpoints are only checked when Inspector is non-null.

public HashSet<int> Breakpoints { get; }

Property Value

HashSet<int>

Remarks

Addresses are raw integers and are not validated against the current program image. Negative, past-end, and mid-operand values are accepted silently and will never be hit unless the instruction pointer happens to equal that exact value during dispatch. Stale breakpoints are not removed when a new program is loaded into the context.

Methods

Execute(T, CancellationToken, long)

Executes all instructions until the end of the data stream, until InstructionPointer becomes negative (program termination signal), until cancellationToken is cancelled, or until maxInstructions instructions have been dispatched.

public void Execute(T context, CancellationToken cancellationToken = default, long maxInstructions = 0)

Parameters

context T

The execution context.

cancellationToken CancellationToken

Token that can stop execution between instructions.

maxInstructions long

Maximum number of instructions to dispatch. When greater than zero, throws InstructionBudgetExceededException if this limit is reached before the program terminates naturally. When zero (the default), execution is unlimited.

Remarks

Cancellation is cooperative and checked only between instructions — not within a single instruction handler. A handler that blocks indefinitely, performs a long-running computation, or enters an infinite loop will prevent cancellation from being observed until it returns. For untrusted or long-running handlers, use an isolation boundary or have the handler observe the token or a deadline directly.

Exceptions

VirtualProcessorException

Thrown on an unknown opcode sequence.

OperationCanceledException

Thrown when cancellationToken is cancelled.

InstructionBudgetExceededException

Thrown when maxInstructions is greater than zero and that many instructions have been dispatched without the program terminating.

Human-readable name used in diagnostics. Must not be null, empty, or whitespace.

Delegate invoked when the opcode is matched.

overwrite bool

When true, replaces an existing registration for the same opcode. When false (default), throws if the opcode is already registered.

Exceptions

Thrown when name is null.

ArgumentException

Thrown when opcode is empty, when name is empty or whitespace-only, or when the opcode is already registered and overwrite is false.

ValidateCompletion(T)

Called by Scheduler<T> immediately before a process transitions to Terminated due to normal program completion (instruction pointer out of range or negative). Throw to fail-fast and transition the process to Faulted instead.

public virtual void ValidateCompletion(T context)

Parameters

context T

The execution context at termination.

Remarks

Override this method to validate that the context is in a structurally consistent terminal state: call frames are empty, control-flow blocks are all closed, no exception handler is pending, operand stacks are empty, etc. The default implementation does nothing, maintaining backward compatibility.
