Represents a WKS (Well Known Services) record, as specified by RFC 1035 §3.4.2. A WKS record details which well-known services are available for a given IPv4 address and IP protocol (e.g., TCP or UDP).

Remarks

The WKS record includes:

  • A 32-bit IPv4 address (IPAddress)
  • An 8-bit protocol number (Protocol). Examples: 6 (TCP), 17 (UDP).
  • A variable-length bit map (Bitmap), with one bit per port. For instance, if Protocol=6 (TCP), the 26th bit corresponds to TCP port 25 (SMTP). If that bit is set, the service on port 25 is advertised as available.

For each bit in Bitmap, the index corresponds to a port number. If the bit is 1, that port is available for the specified Protocol; otherwise 0 means unavailable. This approach is considered obsolete and rarely used in modern DNS contexts. Instead, SRV records or other service discovery methods are typically employed.

Example usage (pseudocode): 

var wksRecord = new WKS
{
    IPAddress = IPAddress.Parse("192.0.2.1"),
    Protocol = 6, // TCP
    Bitmap = new byte[] { 0x00, 0x01, ... } // Indicating ports 8, 9, etc. might be active
};

WKS RRs do not trigger additional section processing. They are primarily for describing availability of TCP/UDP services on an IPv4 address, although they are considered effectively deprecated in modern DNS practice.

The on-the-wire layout (RFC 1035 §3.4.2) is:

+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                    ADDRESS                    |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|       PROTOCOL        |                       |
+--+--+--+--+--+--+--+--+                       |
|                                               |
/                   <BIT MAP>                   /
/                                               /
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+

ADDRESS is the IPv4 address, PROTOCOL is the IP protocol number, and the bit map advertises active ports (nth byte covers ports n*8 to n*8+7).
