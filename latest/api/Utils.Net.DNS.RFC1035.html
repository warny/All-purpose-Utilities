Namespace Utils.Net.DNS.RFC1035

Classes

Address

Represents an IP address record (A, AAAA, or NSAP) in DNS, as described by RFC 1035 §3.4.1 (A records), RFC 1886 (superseded but historically defining AAAA records), and the OSI NSAP address type. This class stores the IP address and reflects the correct numeric record ID based on the address family.

CNAME

Represents a DNS CNAME (Canonical Name) record, used to alias one domain name to another, as defined in RFC 1035 §3.3.1.

Default

Represents a fallback or "default" DNS record for scenarios where the record type is unrecognized, or when a custom processing path is desired for type code zero (which is reserved/unused). Despite the TXT-like comment, this class is annotated with [DNSRecord(DNSClass.IN, 0x00)], indicating it is treated as a placeholder for type 0. The raw payload is stored in the Datas property.

HINFO

Represents an HINFO (Host Information) record, as defined in RFC 1035 §3.3.2.

MB

Represents an MB (Mailbox) DNS record, as described in RFC 1035 §3.3.3 (marked as experimental).

MD

Represents an MD (Mail Destination) record, which is an obsolete DNS record type as per RFC 1035 §3.3.4.

MF

Represents an MF (Mail Forwarder) record, an older mail-routing DNS record type now considered obsolete per RFC 1035 §3.3.5. Similar to MD, the MF record has been superseded by MX.

MG

Represents an MG (Mail Group) record, classified as experimental in RFC 1035 §3.3.6.

MINFO

Represents a MINFO record, considered experimental and largely obsolete as per RFC 1035 §3.3.7. The MINFO record provides mail-related information (responsible mailbox, error mailbox) for a mailing list or mailbox owner.

MR

Represents an MR (Mail Rename) record, marked as experimental and obsolete as per RFC 1035 §3.3.8.

MX

Represents an MX (Mail Exchange) record in the DNS, as described in RFC 1035 §3.3.9 and further clarified by RFC 974. The MX record specifies a mail server responsible for accepting email on behalf of a domain, along with a preference value.

NS

Represents an NS (Name Server) record as specified in RFC 1035 §3.3.11. An NS record indicates the authoritative name server for a given domain.

NULL

Represents a NULL (experimental) DNS record as described in RFC 1035 §3.3.10. This record type can hold arbitrary data (up to 65535 bytes) and is primarily a placeholder for experimental or interim purposes.

PTR

Represents a PTR (Pointer) record in DNS, as specified by RFC 1035 §3.3.12.

SOA

Represents an SOA (Start of Authority) record, which defines zone-wide parameters including the primary name server, mailbox of the zone administrator, and various time intervals affecting caching and zone transfers, as described in RFC 1035 §3.3.13.

TXT

Represents a TXT record in DNS, as defined in RFC 1035 §3.3.14. TXT records store arbitrary human-readable text, often used for descriptive or application-specific data (e.g., SPF, DKIM, etc. in modern usage).

WKS

Represents a WKS (Well Known Services) record, as specified by RFC 1035 §3.4.2. A WKS record details which well-known services are available for a given IPv4 address and IP protocol (e.g., TCP or UDP).
