GetAllBinFrequencies(Complex[], double)

Returns the frequency represented by every bin of a Fourier transform result, including the negative (aliased) frequencies of the upper half - matching GetAmplitudes(Complex[])/ GetPhases(Complex[])'s bin count exactly, unlike GetFrequencies(Complex[], double) (see TODO-2026-07-11-pass4.md item #54).

public static double[] GetAllBinFrequencies(this Complex[] transform, double sampleRate)

Parameters

transform Complex[]

The transform result. Must be non-null and non-empty.

sampleRate double

Sampling rate in hertz. Must be finite and positive.

Returns

double[]

Array of N frequencies in hertz: bin i < ceil(N/2) is the positive frequency i * sampleRate / N; bin i >= ceil(N/2) is the negative (aliased) frequency (i - N) * sampleRate / N - the same convention as e.g. NumPy's fftfreq, under which an even-length transform's Nyquist bin is reported as the negative Nyquist frequency.

Exceptions

ArgumentNullException

Thrown when transform is null.

ArgumentException

Thrown when transform is empty.

ArgumentOutOfRangeException

Thrown when sampleRate is not finite or is not positive.

GetAmplitudes(Complex[])

Returns the raw spectral magnitude (Magnitude) of each bin in the transform result - not a physically normalized sinusoidal amplitude (see TODO-2026-07-11-pass4.md item #55). Depending on the forward transform's own normalization convention, signal length, one-sided-vs-two-sided selection, and any window's coherent gain, the value that corresponds to a signal component's actual physical amplitude generally requires further scaling by the caller (e.g. this library's own forward transform is unnormalized, so a pure sinusoid's peak bin magnitude equals signal amplitude * N / 2, not the amplitude itself - see GetMagnitudes(Complex[])'s "constant signal" example). GetMagnitudes(Complex[]) is an identical, more accurately-named alias; this method is kept for backward compatibility.

GetMagnitudes(Complex[])

Returns the raw spectral magnitude (Magnitude) of each bin in the transform result. The more accurately-named counterpart to GetAmplitudes(Complex[]) (see TODO-2026-07-11-pass4.md item #55): "amplitude" suggests a physically normalized sinusoidal amplitude, but this is the raw, unnormalized FFT bin magnitude - e.g. for this library's unnormalized forward transform, the FFT of a constant signal of value 1 over N samples has a DC-bin magnitude of N, not 1.
