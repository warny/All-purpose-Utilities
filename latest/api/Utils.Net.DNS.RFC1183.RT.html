Remarks

The RT record includes:

  • Preference: A 16-bit integer specifying the relative preference of this route. Lower values indicate more preferred routes.
  • DnsName: The domain name of the intermediate host or router that should be used for routing to the owner name of this record.

Example usage might look like: 

example.com.  IN  RT  10  gateway.example.net.

indicating that traffic to example.com should be routed through gateway.example.net with preference 10. Lower Preference values mean higher priority.

The RT record does not trigger additional section processing (no extra A/AAAA records included by default). In modern networking, explicit routing via DNS is rarely practiced, and the RT record is largely historical.

The wire layout is:

+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                  PREFERENCE                   |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
/                   EXCHANGER                   /
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
