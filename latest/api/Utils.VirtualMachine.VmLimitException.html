Remarks

Derives from VmInvalidOperationException so that limits previously reported as InvalidOperationException (e.g. operand-stack overflow) remain catchable by the same handlers after the migration to the structured limit hierarchy.

Carries the three diagnostic fields common to every limit violation: LimitKind, Limit, and AttemptedValue. Catch handlers may target this type to intercept any limit violation regardless of which specific resource was exhausted.
